01

Course Overview

This course provides a comprehensive introduction to computer systems from a programmer's perspective. Students will explore how computer systems execute programs, store information, and communicate.


The course bridges the gap between high-level programming and the underlying hardware, giving students the knowledge needed to write efficient programs and understand system behavior — from binary representation all the way to operating system fundamentals.

Primary Learning Outcomes

  • Understand how programs are represented and executed at the machine level
  • Master data representation and manipulation techniques
  • Comprehend processor architecture and optimization strategies
  • Analyze memory hierarchy and system performance
  • Understand linking, virtual memory, and OS fundamentals

03

Weekly Schedule

WK 1–2 Course Introduction & Information Representation
  • Course overview and expectations
  • History and evolution of computer systems
  • Binary number systems and arithmetic
  • Integer representation and operations
  • Floating-point representation (IEEE 754)
WK 3–4 Machine-Level Representation
  • Assembly language fundamentals
  • x86-64 instruction set architecture
  • Register usage and calling conventions
  • Control flow and conditional execution
  • Procedures and stack management
WK 5–6 Processor Architecture
  • Sequential processor implementation
  • Pipelined processor design
  • Hazards and forwarding
  • Branch prediction and speculation
  • Superscalar and out-of-order execution
WK 7–8 Program Optimization
  • Compiler optimizations
  • Loop unrolling and vectorization
  • Memory access patterns
  • Cache-friendly programming
  • Profiling and performance measurement
WK 9–10 Memory Hierarchy
  • Storage technologies and trends
  • Cache memory organization
  • Cache performance and optimization
  • Memory mountain and locality
  • Cache-conscious programming techniques
WK 11–12 Linking and Loading
  • Static and dynamic linking
  • Object file formats (ELF)
  • Symbol resolution and relocation
  • Shared libraries and position-independent code
  • Library interpositioning techniques
WK 13–14 Exceptional Control Flow
  • Exceptions, interrupts, and system calls
  • Process creation and termination
  • Signal handling and non-local jumps
  • Process synchronization
  • Inter-process communication
WK 15–16 Virtual Memory & OS Fundamentals
  • Virtual memory concepts
  • Address translation and page tables
  • Memory protection and sharing
  • Operating system overview
  • System calls and kernel interface
  • Course review and final preparation

Course Policies

Attendance Policy

Regular attendance is expected. Missing more than 3 classes may result in a grade reduction. Please inform the instructor in advance if absence is unavoidable.

Late Assignment Policy

Late assignments lose 10% per day. No assignments are accepted more than 3 days after the deadline without prior arrangement.

Academic Integrity

All work must be original. Collaboration on concepts is encouraged, but submitted code must be individually written. Violations will be reported.

Make-up Exams

Make-up exams are available only for documented emergencies with prior notification to the instructor before the exam date.
